# ネイティブ言語
- 基本的に日本語で返答して
- 英語の方が伝わりやすいことなどは英語でok

# 使用プログラミング言語
- Rust,Pythonをメインで使用
- 始めてから1年程度の初心者

# パッケージ管理
- Rust: cargo
- Python: uv

# コード説明
- 初心者なので「なぜそう書くか」の理由も含めて常に説明して
- 新しい構文・概念が出てきたときは特に丁寧に

# コメント
- コード内コメントは日本語で書く

# 依存ライブラリの追加
- 標準的で小さいライブラリは自動で追加してOK
- 主要な依存になるもの・大きいライブラリは提案してから追加

# 不明点の扱い
- 推測で進めず、不明点があれば質問して

# 破壊的操作
- rm -rf、DB削除、force pushなど取り返しのつかない操作は実行前に必ず確認

# 環境
- PC：Mac　たまにWindows
- シェル：Zsh
- エディタ：Neovim
- ターミナルエミュレータ：wezterm

# 出力について
- アプリのプログラムなどについてはそのコードでリリースできるかを確認してから出力して

# git関係
- pushしておいてという時は特別な指示がない限りcommitしてからpushしておいてという指示
- コードを書いた際は適切な粒度でコミットをしておく

# ファイルアクセス
- .sshフォルダにはアクセス不可
- .envには絶対アクセスはしない

# 使用ツール
- マークダウンをpdfにする際はpandocを使用
- ファイル/コード検索は `grep` ではなく `rg` (ripgrep) を使う
- ディレクトリ走査・ファイル列挙は `find` ではなく `rg --files` か `fd` を優先
